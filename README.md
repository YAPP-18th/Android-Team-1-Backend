## 2. git convention 


|명령어|내용|
| :-----------------------------------: | :---------------------------------------: |
| CHORE | 빌드 업무 수정, 패키지 매니저 수정 |
| **ADD** |   코드나 테스트, 예제, 문서 등의 추가   |
| **FIX** | 올바르지 않은 동작을 고친 경우 |
| **REMOVE** |   코드의 삭제가 있을 때   |
| **UPDATE** |   문서나 리소스, 라이브러리등의 수정, 추가, 보완   |
| **FEAT** |  새로운 기능 추가   |
| **CORRECT** | 주로 문법의 오류나 타입의 변경, 이름 변경 등   |
| REFACTOR |   코드의 전면적인 수정   |
| DOCS |  문서의 개정   |
| RENAME | 파일의 이름 변경 |
| TEST | TEST 코드 관련   |
